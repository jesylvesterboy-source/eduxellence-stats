How DataClean Pro Works

11 automated cleaning stages run on your file the moment you click "Clean."

🧹
Whitespace stripping
Column names and all string cells are trimmed of invisible spaces.
🎯
Null marker standardisation
Converts N/A, None, #N/A, -, ., blank → proper null values.
🗑
Blank row / column removal
Drops rows and columns that are completely empty.
🔍
Smart type detection
Auto-identifies numeric, date, categorical, text, boolean, and identifier columns.
📝
Category standardisation
MALE / male / Male (with trailing space) → Male. Consistently.
📅
Date standardisation
8 different date formats (Jan 15 2024, 15/01/2024, 2024-01-15…) → single YYYY-MM-DD standard.
💰
Currency / numeric parsing
$1,200.00 / ₦45,000 / £850 → clean float values for analysis.
👥
Duplicate removal
Exact duplicate rows detected and removed, keeping the first occurrence.
⚠
Outlier flagging
IQR-based detection flags suspicious values in new _outlier_* columns. We flag — you decide.
🩹
Missing value imputation
Numeric columns filled with column median. Categorical columns filled with mode.
📊
Column intelligence report
Every column audited: type, missing %, unique count, min/max/mean. Included in your Excel file.

Frequently Asked Questions

Is this really free?
Yes, completely free. No account, no credit card, no watermark on your output. It is a free tool built by Eduxellence Analytics to help data professionals.
Is my data private?
Your file is processed entirely on the server, used once to generate the clean file, and then deleted immediately. We do not store, read, log, or share your data.
What file types are supported?
CSV (.csv), Excel 2007+ (.xlsx), and older Excel (.xls). Maximum file size is 10 MB. For larger files, use the downloadable Python version.
What does the cleaned Excel file contain?
Four worksheets: (1) your cleaned data, (2) a full cleaning log of every action taken, (3) a column-by-column intelligence report, and (4) run statistics.
What if the tool gets something wrong?
The script uses smart heuristics but is not perfect. Always review the cleaning log sheet and the _outlier_* columns before using the data in a model.
